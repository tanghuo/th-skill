# 业务状态机 & 不变量

> 合并原"业务状态机 & 非法状态流转"和"业务不变量/守恒关系"。

```text
梳理项目核心业务状态机和关键不变量，检查是否有非法流转或不变量被破坏的路径。

状态机检查：

- 是否存在未定义状态或废弃状态仍被使用
- 是否允许从非法前置状态迁移到目标状态
- close / cancel / expire / finish / settle 终态是否还能被修改
- 状态流转是否缺少幂等判断
- 不同入口是否绕过统一状态机直接改字段
- MySQL、Redis、内存、前端展示状态含义是否一致
- 定时任务和用户操作是否可能同时修改状态
- 状态字段和时间字段是否绑定一致（如 closed_at 有值但 status 仍 running）

业务不变量检查：

- 金额守恒：收入、支出、余额、结算、手续费是否对齐
- 唯一性：一个用户只能有一个活跃房间 / 任务 / 绑定
- 生命周期：终态对象不能再被修改
- 计数守恒：成员数、在线数、热度、礼物数是否和来源一致
- 时间不变量：开始、结束、过期、结算时间顺序是否合法
- 幂等：同一请求/回调/任务重复执行结果一致

每个发现给出：

- 不变量或合法状态流转
- 哪条路径破坏它
- 最小复现顺序
- 业务后果
- 修复方向
```
