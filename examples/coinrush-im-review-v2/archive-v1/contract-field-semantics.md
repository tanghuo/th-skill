# 字段单位、语义 & 时间

> 合并原"字段单位与语义"和"时间语义/时区/TTL"。

```text
检查金额、热度、时间、计数、状态字段的单位和语义是否在各层保持一致。

单位与语义：

- 分/元/金币/钻石/积分是否混用
- 秒/毫秒/纳秒是否混用
- 内部存储单位和对外展示单位是否混用
- 文档字段说明是否和实际计算公式一致
- DB 字段名与业务含义是否已不匹配
- 历史字段是否被新业务复用导致语义变化

时间：

- UTC、本地时区、服务器时区是否混用
- Unix timestamp 与 DB datetime 转换是否一致
- Redis TTL 与 MySQL 过期时间是否一致
- closed_at 有值但状态未关闭，或状态关闭但时间字段为空
- 定时任务按自然日/自然小时/滚动窗口/UTC 的边界是否和产品定义一致
- `<` / `<=` 边界是否导致重复扫描或漏扫描

只报告会导致前端展示错误、统计错误、结算错误、风控错误或兼容性问题的差异。
```
