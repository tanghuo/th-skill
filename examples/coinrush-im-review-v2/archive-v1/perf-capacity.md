# 性能退化 & SQL / 索引

> 合并原"性能退化 & 容量风险"和"SQL/索引专项"。

```text
扫描高频入口、批处理、定时任务、WebSocket 广播、RPC 和数据库查询路径，找出数据量或并发量上升后会明显退化的问题。

应用层：

- 循环中查 DB / Redis / RPC，形成 N+1
- 未分页读取大表、大集合、大 Redis key
- Redis KEYS / SMEMBERS / ZRANGE 全量读取等无界操作
- 大 slice / map / JSON 全量构造或复制
- WebSocket 广播被单个慢连接阻塞
- goroutine / channel / 队列无上限
- 批处理扫全表、重复扫、无法恢复进度
- 热路径重复解析配置、构建客户端
- 高并发路径使用全局锁

SQL / 索引：

- where 条件无法命中索引
- order by 无法利用索引
- limit + offset 深分页
- like 前缀通配、函数包裹索引字段、隐式类型转换导致索引失效
- 复合索引字段顺序与查询不匹配
- SELECT *、批处理无分页
- 更新或删除缺少限制条件

每个发现说明：

- 入口路径和数据规模增长方式
- 退化点
- 最坏情况影响
- 修复方向：分页、索引、批量查询、异步化、限流、背压、缓存或增量扫描
```
