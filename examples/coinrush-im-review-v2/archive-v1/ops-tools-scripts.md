# 运维工具 & 脚本 & 启动顺序

> 合并原"一次性工具/脚本"和"运行时配置与启动顺序"。

```text
检查 tools/、scripts/、临时清理程序、数据修复脚本和服务启动配置是否安全。

工具与脚本：

- 默认连接哪个环境
- dry-run 是否真的不写数据
- destructive 参数是否需要二次确认
- 批量删除/更新是否有分页、限速和失败恢复
- 是否打印用户隐私或敏感响应
- where 条件为空是否有保护
- 失败后是否留下半更新状态

启动与配置：

- 必需依赖是否在构造或启动阶段 fail-fast
- 业务逻辑里是否用 nil guard 静默跳过必需能力
- 配置默认值是否会误连错误环境
- 配置缺失时是否使用危险默认值
- dev / test / prod 配置是否可能混用
- 初始化失败后服务是否仍注册 ready

每个发现说明：

- 入口和默认行为
- 危险参数或配置
- 误操作后果
- 防护建议
```
