# 金额精度 & 溢出

> 礼物 / 充值 / 结算 / 余额路径专项。
> 与 `consistency-state-machine` 的"金额守恒"互补：守恒看账目对齐，本文件看单值的精度、溢出、符号。

```text
扫描涉及金额、礼物价值、余额、积分、手续费、分成的计算与存储路径，检查精度、溢出和符号问题。

精度与类型：

- 金额是否用浮点（float / double）参与计算或存储，存在精度丢失
- 整数分 / 厘 与展示元之间换算是否一致，是否有 *100 / /100 散落各处口径不一
- 不同字段单位是否混用（分 vs 元 vs 厘），跨表 / 跨接口传递是否换算
- DB 列类型（DECIMAL 精度位、BIGINT、INT）能否容纳业务最大值
- 比例 / 分成 / 折扣计算的取整方向（floor / round / ceil）是否明确且一致，多方分账是否对齐总额

溢出与边界：

- 金额累加（流水合计、礼物总值、热度）是否可能超过 INT / BIGINT 上限
- 充值倍率、礼物数量 × 单价是否可能溢出
- 是否允许负数金额 / 负数数量进入计算，校验是否在入口
- 余额扣减是否校验充足，是否可能扣成负数
- 数量为 0 或极大值时计算是否仍合法

并发与一致性：

- 余额 / 计数扣减是否用原子操作或乐观锁，并发下是否会超扣 / 漏扣
- 扣款与发货 / 入账是否在同一事务或有幂等键（参考 consistency-atomicity）

每个发现说明：

- 涉及的金额字段和计算路径
- 精度丢失 / 溢出 / 负值 / 超扣 的具体触发输入
- 资损方向（多扣用户 / 少结算 / 平台亏损）
- 最小修复方向（改 DECIMAL / 整数分 / 入口校验 / 原子扣减）
```
