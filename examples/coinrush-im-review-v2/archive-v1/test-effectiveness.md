# 测试有效性

```text
审计测试是否覆盖核心业务不变量、错误分支、并发路径、幂等路径和状态恢复路径。

重点关注：

- 只有 happy path，没有失败分支
- 测试只断言返回值，不断言 MySQL / Redis / WS / MQ 副作用
- 使用 mock 后绕过了真实事务、序列化、权限或状态机
- 并发创建、重复回调、重复关闭、重复结算没有测试
- 测试依赖执行顺序、时间、真实外部服务或共享全局状态
- bug 修复没有对应回归测试
- 关键业务不变量没有被测试表达
- 测试没有覆盖迁移兼容、旧数据、旧客户端 payload
- 测试没有验证权限边界和对象归属

只报告会导致重要问题无法被测试发现的缺口，不报告单纯覆盖率不足。
```
