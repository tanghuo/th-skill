# 可观测性缺口

```text
检查核心业务链路的日志、指标、trace 和告警是否足以定位线上问题。

重点关注：

- 关键动作（创建、关闭、结算、回调、重试、推送）是否有结构化日志
- 日志是否包含 request_id / trace_id / user_id / room_id 等关联字段
- error log 是否缺少关键上下文
- 关键状态变化没有业务指标
- metrics label 是否存在高基数风险（user_id、token、完整 URL）
- 批处理跳过异常数据但没有计数或告警
- 幂等冲突、重复 callback、非法状态流转没有指标
- 关键异步任务失败后没有告警

只报告会影响线上定位、故障恢复或事故追踪的问题，不报告普通 debug 日志不足。
```
