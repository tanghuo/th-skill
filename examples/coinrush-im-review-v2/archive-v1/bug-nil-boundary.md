# 边界条件 & nil

```text
扫描所有从 map / slice 取值、类型断言、指针解引用、JSON 反序列化、RPC 返回、Redis 查询结果中直接使用值的位置，找出缺少 nil / ok / len 检查的问题。

重点关注：

- RPC 返回值、Redis 查询结果、JSON 反序列化后的字段
- map 取值后直接使用零值（无法区分"不存在"和"值为零"）
- slice 下标访问、类型断言未检查 ok
- 指针字段直接解引用
- 可选字段、历史数据字段、新增字段
- 金额、计数、状态、权限、配置开关的零值语义是否危险
- nil guard 掩盖必需依赖缺失导致静默降级

只报告会导致 panic、权限错误、金额错误、状态错误、展示错误或兼容性问题的情况。
```
