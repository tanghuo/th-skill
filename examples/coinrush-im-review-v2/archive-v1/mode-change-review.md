# 变更 Review 模式

> 用于 PR、diff、最近提交或功能分支的增量审计。  
> 使用时替换 `_base.md` 中的通用描述，不需要同时拼接 `_base.md`。

```text
你现在做变更 Review，只审计当前变更引入的问题，不审计历史遗留。

请先识别本次变更涉及的：

- 新增入口 / 修改的写路径
- 修改的状态字段 / 外部契约 / 配置项
- 修改的定时任务或 callback
- 修改的 Redis key / schema / model
- 修改的权限判断 / 脚本 / 部署配置

然后只报告由本次变更直接引入或明显放大的问题。
不要报告与本次变更无关的历史问题，除非本次变更会触发它。

严重度：P0=资损/泄漏/不可用  P1=重复结算/越权/不一致  P2=特定条件异常/语义错误  P3=低概率边界/技术债

每个发现说明：

- 为什么是本次变更引入或放大的
- 变更前后行为差异
- 是否影响兼容性、状态一致性、幂等、安全或性能
- 最小修复方向

忽略测试文件、生成代码、vendor。
如果没有发现问题，输出"未发现变更引入的问题"并列出检查范围。
```
