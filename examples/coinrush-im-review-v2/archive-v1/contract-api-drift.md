# 文档 / API / 代码漂移

```text
检查项目中面向前端或外部系统的契约是否一致。

关注来源（可信度从高到低）：
1. 实际线上入口和返回结构
2. signal.api / protobuf / OpenAPI
3. docs/*.md
4. 前端消费示例
5. 注释

当来源冲突时，不默认文档正确，需说明哪侧更可信。

重点找出：

- 字段名、类型、required/optional、默认值、enum 含义不一致
- 文档说会返回但代码没返回，或反之
- HTTP 与 WS 表达同一语义但字段不同
- callback 文档与实际处理逻辑不一致
- 错误码定义不一致

只报告会导致前端、外部系统、兼容性或业务语义出错的问题。
```
