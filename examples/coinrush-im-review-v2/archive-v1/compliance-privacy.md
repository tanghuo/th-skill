# 隐私 & 合规 & 依赖安全

> 合并原"隐私数据/合规工程风险"和"依赖/构建/CI/镜像安全"。  
> 低频使用，按需审计。

```text
从工程实现角度检查敏感数据处理和依赖供应链安全。

隐私与合规：

- 敏感字段是否进入日志、错误返回、埋点、调试接口或第三方请求
- 用户删除/注销/封禁后相关数据是否仍被使用
- 数据是否有保留期限和清理任务
- 导出、补偿、修复脚本是否打印或保存敏感数据
- 前端可见字段是否包含内部风控、结算、权限或调试信息
- 数据删除是否只删主表，未删缓存、日志、对象存储或索引

依赖安全：

- go.mod 中 replace 到本地路径或临时 fork
- 依赖版本长期未更新或存在已知漏洞
- Docker base image 过旧、使用 latest tag
- CI 中泄漏 token、secret 或生产配置
- 构建产物包含 .env、私钥、测试数据

不要给法律结论，只报告工程实现上的合规和供应链风险。
```
