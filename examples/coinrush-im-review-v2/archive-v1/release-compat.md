# 发布兼容性 & Schema 迁移

```text
检查数据库迁移、配置变更、API 变更、消息格式变更、Redis key 变更和部署脚本，确认是否支持真实生产发布、灰度和回滚。

重点关注：

- 新代码依赖新字段，但迁移不保证先执行
- 新字段没有默认值，历史数据语义不明确
- 删除或改名字段后，旧版本服务仍可能读取
- API / WS / protobuf 字段变更不兼容旧客户端
- Redis key 格式变更没有兼容旧格式
- 消息队列 payload 变更导致旧消费者失败
- 回滚后新数据无法被旧代码识别
- migration 中有长事务、全表锁、无分页更新等风险
- 字段从 nullable 改 non-nullable 是否兼容历史数据
- 多服务部署顺序没有在脚本或 compose 中真实保证

只报告会影响生产发布、灰度、回滚、旧客户端兼容或历史数据处理的问题。
```
