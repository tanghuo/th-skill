# 事务边界 & 幂等 & 半成功状态

> 合并原"并发写入与幂等"、"事务边界 & 半成功状态"、"异步任务/重试/顺序性"三个主题。

```text
检查项目中逻辑上应该原子完成的业务路径，确认 MySQL、Redis、RPC、第三方 API、MQ、WS 推送和内存状态之间是否存在半成功状态或缺乏幂等边界。

重点关注：

事务边界：
- 一个业务动作多次 DB 写入但无事务
- 事务内调用外部 RPC / HTTP / Redis / 发送消息
- MySQL 写成功后 Redis / MQ / WS 推送失败无补偿
- Redis 标记成功但 MySQL 回滚
- 事务提交前发通知导致外部看到未提交状态
- 事务中执行耗时外部调用导致锁持有时间过长

幂等：
- 重复请求或重复回调是否重复写入
- 唯一键 / 事务 / 锁 / Redis lease 是否和业务不变量一致
- 状态终态是否还能被重复修改
- 幂等键是否包含正确业务维度
- 乐观锁 / CAS 失败后是否正确处理
- 分布式锁过期后是否可能双写

异步任务：
- 同一业务对象是否可能被多个 worker 同时处理
- 任务重试是否重复执行副作用
- 批处理扫到异常数据后是修复/清理/标记还是永远重复扫描
- callback 到达顺序是否可能乱序
- worker 退出时是否丢弃内存中的任务
- 批处理是否有分页、游标、checkpoint

每个发现给出：
- 真实入口和操作顺序
- 失败点和半成功状态
- 缺失的幂等边界
- 最小复现顺序
- 业务后果
- 修复方向：事务、outbox、状态机、补偿、幂等键、重试去重或顺序调整
```
