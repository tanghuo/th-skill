# 测试有效性

> 审计测试能不能拦住重要回归，不是看覆盖率。
> 本机跑测试需 `GOMODCACHE=/tmp/gomodcache GOCACHE=/tmp/gocache go test ...`（默认 cache 不可写）。

```text
审计测试是否覆盖核心不变量、错误分支、幂等路径和并发路径。

起点：
- 现有测试分布：internal/job 下的 *_test.go（结算/修正类已有测试），对照哪些核心路径完全无测试
- 高价值缺口优先：结算重跑、重复回调、重复关房、并发送礼、权限边界

重点：
- 只有 happy path，无失败分支；只断言返回值，不断言 MySQL/Redis 副作用
- mock 绕过了真实事务、状态机或权限判断，测试通过但保护为零
- 重复结算、重复回调、并发创建等幂等场景无测试
- 测试依赖执行顺序、真实时间、外部服务或共享全局状态（不稳定即无效）
- bug 修复无对应回归测试；关键业务不变量（金额守恒、终态不可改）没有被任何测试表达
- 迁移兼容、旧数据、旧客户端 payload 无测试

只报告会让重要问题漏网的缺口，并按"该缺口对应的最严重事故"排序。每个发现说明：缺口、对应风险场景、建议补的最小测试（用例描述即可，不写实现）。
```
