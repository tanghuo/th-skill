# 质量审计提示词库 v2

> v1（5.5pro 生成版）已整体归档至 `archive-v1/`。
> v2 重构要点：21 主题合并为 15；每个主题带本仓库真实起点锚点和已知业务语义（防误报）；`_base` 强制"读过代码才能报告"+ 数量纪律；新增运行记录表跟踪验证状态。

## 使用方式

- **Claude Code（推荐）**：`/audit <主题>` 自动拼接执行，如 `/audit money`、`/audit arch A`、`/audit concurrency apps/gateway/`。审计完成后更新下方运行记录表。
- **Codex `/goal`**：手动拼接 `_base.md` + 一个主题文件。
- 纪律：一次只跑一个主题；在干净会话中跑；限定范围时在主题前加一行 `只扫描 <路径>`。
- `mode-*` 独立模式不拼接 `_base.md`。

### 独立模式

| 文件 | 用途 |
|---|---|
| `mode-change-review.md` | 变更 Review（PR/diff 增量，要求读 diff 外完整上下文） |
| `mode-over-implementation.md` | 过度实现 Review（diff 相对最小充分解是否做多了：投机性扩面/过度防御/抽象/配置/测试/注释） |
| `mode-entry-map.md` | 高风险入口识别，产物落盘 `entry-map.md` 可增量刷新 |

## 推荐节奏

- **首次**：`mode-entry-map` 产出地图 → 按主题表顺序跑前 5 个
- **月度**：跑 2-3 个主题，选择依据：近期改动集中区 + 运行记录里最久未跑的
- **PR**：`mode-change-review`，可追加一个主题深挖
- **发布前**：`release-ops` → `security-exposure`
- **架构**：每 1-2 个月 `/audit arch`，按 `mode-architecture-review.md` 的账本机制走
